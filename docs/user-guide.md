## Property 店主指南

### 第一章：开设您的数字店面

#### 1.1 什么是 Property Studio？

Property Studio 是您在 OpenStay 网络中的**独立数字店面**。与 OTA 平台不同：

- ✅ **品牌属于您** - 突出您的店名和故事，而非平台
- ✅ **数据属于您** - 客户信息、订单数据完全自主
- ✅ **多渠道分销** - 可同时加入多个 Hub，不被单一平台绑定
- ✅ **主题自定义** - 可选择或设计独特的页面风格

#### 1.2 快速开始

**步骤 1: 部署 Blocklet**

```bash
# 通过 Blocklet Store 部署
# 或使用 CLI 工具
blocklet create my-property --template openstay-property
```

**步骤 2: 基础配置**

1. 使用 DID 钱包登录管理后台
2. 填写旅店基本信息：
   - 店名、地址、联系方式
   - 入住/退房时间
   - 设施标签
3. 上传营业执照（用于 Hub 审核）
4. 质押 Node NFT 激活 Property

**步骤 3: 录入房源**

1. 创建房型：
   - 房型名称（如「海景大床房」）
   - 面积、床型、最大入住人数
   - 设施详情
   - 高质量图片（建议 5-10 张）

2. 设置日历与价格：
   - 基础价格
   - 周末/节假日价格规则
   - 可售房态

**步骤 4: 加入 Hub**

1. 浏览可用的 Hub 列表
2. 选择适合的 Hub（如区域 Hub、垂类 Hub）
3. 提交入驻申请
4. 等待审核通过
5. 房源自动同步到 Hub

#### 1.3 日常运营

**订单管理**

- **新订单通知**: 邮件/短信/App 推送
- **确认订单**: 24 小时内确认，避免自动取消
- **拒单**: 填写原因，系统自动通知客人
- **改期改价**: 与客人协商，双方确认后生效

**日历维护**

- 定期更新房态，避免超售
- 节假日提前调价
- 维护期间标记「关房」

**评价回复**

- 及时回复客人评价
- 积极解决客人提出的问题
- 良好的评价有助于在 Hub 中获得更好排名

#### 1.4 直连模式

除了通过 Hub 获客，您还可以通过**直连模式**直接接待客人：

**使用场景**:
- 老朋友推荐
- 社交媒体分享
- 官网 SEO 流量

**操作方式**:
1. 在管理后台生成分享链接/二维码
2. 分享至微信/微博/小红书
3. 客人扫码直达您的页面
4. 页面显示「⚠️ 未经 Hub 审核」提示，客人自担风险

**优势**: 无佣金、无中间商
**注意**: 请确保页面信息真实，维护自身信誉

## Hub 运营者指南

### 第二章：创建您的聚合平台

#### 2.1 什么是 Hospitality Hub？

Hub 是 OpenStay 网络中的**聚合与审核平台**，类似于「数字商场」或「旅游集散中心」：

- 🔍 **房源发现** - 帮助旅客找到合适的住宿
- ✅ **质量审核** - 对入驻 Property 进行资质审查
- 🌟 **特色运营** - 打造区域或垂类品牌（如「云南精品民宿联盟」）
- 💰 **商业模式** - 通过订阅费 + 订单分成盈利

#### 2.2 Hub 类型建议

| 类型 | 定位 | 示例 |
|------|------|------|
| **区域 Hub** | 覆盖特定地理区域 | 「大理民宿联盟」「京都传统旅馆协会」 |
| **垂类 Hub** | 专注特定住宿类型 | 「潜水者之家」「滑雪度假村联盟」 |
| **主题 Hub** | 围绕特定主题 | 「设计师酒店精选」「亲子友好住宿」 |
| **企业 Hub** | 服务商务差旅 | 「科技公司差旅优选」 |

#### 2.3 创建 Hub

**步骤 1: 部署 Hub Blocklet**

```bash
blocklet create my-hub --template openstay-hub
```

**步骤 2: 基础配置**

1. 设置 Hub 名称和定位
2. 配置审核标准（资质要求、图片质量等）
3. 设置订阅费用（Basic/Pro/Premium 三档）
4. 配置佣金比例

**步骤 3: 质押激活**

- 铸造 Hub Node NFT
- 质押一定数量的代币
- 激活后出现在链上发现列表

#### 2.4 审核流程

**入驻申请处理**

1. 收到 Property 申请
2. 检查 Node NFT 质押情况
3. 审核资质文件：
   - 营业执照/旅馆业许可
   - 实际图片与描述是否一致
   - 视频验证（可选）
4. 做出决定：
   - **批准**: 选择 tier，开始同步数据
   - **拒绝**: 说明原因，Property 可修改后重新申请
   - **要求补充材料**: 待补充后重新审核

**分级标准建议**

| Tier | 要求 | 权益 | 费用 |
|------|------|------|------|
| **Basic** | 基础资质审核 | 基础列表展示 | 低月费 |
| **Pro** | 视频验证、图片质量检查 | 优先展示、推荐位 | 中等月费 |
| **Premium** | 实地考察、高标准 | 首页推荐、专题合作 | 高月费 |

#### 2.5 运营策略

**内容运营**

- 定期策划专题（如「春日赏花住宿推荐」）
- 优质 Property 故事包装
- 用户 UGC 内容征集

**流量获取**

- SEO 优化
- 社交媒体运营
- 与 KOL/KOC 合作
- 线下活动联动

**数据驱动**

- 分析搜索热词，调整房源结构
- 追踪转化率，优化展示逻辑
- 监控评价质量，及时介入纠纷

## 旅客使用指南

### 第三章：发现理想住宿

#### 3.1 两种发现方式

**方式一：通过 Hub（推荐）**

适合：不确定住哪里，希望有保障的旅客

1. 打开 OpenStay App
2. 选择您信任的 Hub（如「大理民宿联盟」）
3. 在 Hub 内搜索、筛选、比较
4. 列表中的 Property 已通过 Hub 审核，多一份安心

**方式二：直接访问 Property**

适合：已有目标住宿，朋友推荐

1. 扫描店家分享的二维码
2. 或点击朋友分享的链接
3. 直达 Property 页面
4. 注意⚠️风险提示，自行判断

#### 3.2 搜索与筛选

**关键词搜索**

- 城市/区域：「大理」「京都」「普吉岛」
- 地标附近：「故宫附近」「海滩边」
- Property 名称：直接搜索店名

**筛选条件**

- 📅 入住/离店日期
- 👥 入住人数
- 💰 价格范围
- 🏷️ 设施标签（WiFi、空调、停车位等）
- 🌟 评分

**排序方式**

- 推荐（Hub 算法排序）
- 价格从低到高/从高到低
- 评分从高到低
- 距离（需授权位置）

#### 3.3 预订流程

1. **查看详情**
   - 浏览图片、设施、规则
   - 查看位置地图
   - 阅读历史评价

2. **选择日期**
   - 查看日历，选择可用日期
   - 注意最少连住要求
   - 查看取消政策

3. **填写信息**
   - 使用 DID 钱包登录（或游客模式）
   - 填写入住人信息
   - 特殊需求备注

4. **确认订单**
   - 核对订单信息
   - 确认价格明细
   - 阅读并同意入住规则

5. **支付**
   - 选择支付方式
   - 完成支付
   - 收到确认通知

#### 3.4 订单管理

**查看订单**

- 待确认
- 已确认
- 入住中
- 已完成
- 已取消

**退改政策**

- **灵活**: 入住前 1 天免费取消
- **中等**: 入住前 3 天免费取消
- **严格**: 入住前 7 天免费取消，之后部分退款

**遇到问题？**

1. 首先联系 Property 店主协商
2. 无法解决？向入驻的 Hub 投诉
3. Hub 会介入调查，必要时下架 Property

#### 3.5 评价系统

**撰写评价**

- 入住完成后可评价
- 评分维度：卫生、位置、服务、性价比
- 支持文字 + 图片
- 仅真实入住订单可评价（防刷单）

**评价作用**

- 帮助其他旅客决策
- 影响 Property 在 Hub 中的排名
- 优质评价可获 Hub 奖励

## 常见问题

### Q1: 什么是 DID 钱包？我需要吗？

**A**: DID（去中心化身份）钱包是您在 OpenStay 网络中的数字身份证。

- **Property 店主**: 必需，用于管理店面和接收订单
- **Hub 运营者**: 必需，用于运营平台
- **旅客**: 可选，游客模式可浏览，预订建议登录以获得更好体验

推荐钱包: [DID Wallet](https://didwallet.io)

### Q2: Hub 审核有什么标准？

**A**: 不同 Hub 标准不同，常见审核项：

- 营业执照/旅馆业许可
- 实际房源与描述一致性
- 图片质量与真实性
- 消防安全合规
- 卫生标准

查看 Hub 页面可了解具体审核标准。

### Q3: 佣金和费用是多少？

**A**: 费用由 Hub 自行设定，一般包括：

- **订阅费**: Property 向 Hub 支付的月费/年费（按 tier）
- **订单分成**: 通过 Hub 带来的订单，抽取 5-15%
- **直连订单**: 无佣金

具体费率请在加入 Hub 前确认。

### Q4: 我的数据安全吗？

**A**: OpenStay 采用多重安全措施：

- DID 身份验证，减少虚假账号
- 敏感数据加密存储
- 传输层 TLS 加密
- 最小化数据收集原则
- 您可随时导出或删除数据

### Q5: 如何解决纠纷？

**A**: 三级处理机制：

1. **协商**: 旅客与 Property 直接沟通
2. **Hub 介入**: 通过 Hub 投诉，Hub 调查调解
3. **链下仲裁**: 严重纠纷可寻求法律途径

建议保留所有沟通记录和证据。

### Q6: 可以在多个 Hub 同时入驻吗？

**A**: **可以！** 这是 OpenStay 的核心优势之一。

您可以同时加入：
- 「大理民宿联盟」
- 「云南精品住宿」
- 「中国民宿推荐」

多渠道获客，不依赖单一平台。
